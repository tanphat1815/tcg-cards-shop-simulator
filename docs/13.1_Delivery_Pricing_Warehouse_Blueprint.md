## Luồng tổng thể (End-to-End Flow)

```
[1] CartSidebar bấm "Thanh toán"
  → cartStore.checkout()
    → statsStore.spendMoney(total)         ← Trừ tiền ngay
    → deliveryStore.scheduleDelivery(snapshot) ← Đẩy vào queue

[2] Phaser update loop (mỗi frame)
  → DeliveryManager.trySpawnNext()
    → deliveryStore.consumeDelivery()      ← Lấy từng item ra khỏi queue (FIFO)
    → spawnBox()                           ← Tạo Rectangle vật lý tại Delivery Zone

[3] Player tiếp cận box, bấm [F]
  → DeliveryManager.checkPickup()
    → deliveryStore.pickUpBox()           ← Lưu carried box vào store
  → Box theo player, Physics tắt

[4] Player đến kệ, bấm [E]
  → MainScene.handlePlayerInteraction()
    → deliveryManager.handleShelfInteraction(shelfId)
      → Đọc shelfRole từ furnitureStore.placedShelves[shelfId].role
      ┌── role === 'selling':
      │     → Điền vào inventory tier của kệ bán
      │     → deliveryStore.openSetPrice()  ← Popup định giá xuất hiện
      └── role === 'storage':
            → Điền vào storedItems của kệ kho
            → Không openSetPrice           ← Popup KHÔNG xuất hiện

[5] SetPriceModal (chỉ khi role = 'selling')
  → Player điều chỉnh giá
  → deliveryStore.closeSetPrice()
  → inventoryStore.shopItems[itemId].sellPrice = customPrice
```

### 2B. Hàm `npcTakeItemFromSlot` — thêm guard kiểm tra role

```typescript
npcTakeItemFromSlot(shelfId: string) {
  const shelf = this.placedShelves[shelfId]
  if (!shelf) return null

  // NPCs chỉ mua từ kệ bán hàng (selling)
  // ⚠️ Đây là rule quan trọng: storage shelf immune với NPC
  if (shelf.role === 'storage') return null

  const filledTiers = shelf.tiers.map((t, idx) => ({ t, idx }))
    .filter(x => x.t.itemId && x.t.slots.length > 0)
  if (filledTiers.length === 0) return null

  const picked = filledTiers[Math.floor(Math.random() * filledTiers.length)]
  const tier = picked.t
  const itemId = tier.itemId!

  tier.slots.pop()
  if (tier.slots.length === 0) {
    tier.itemId = null
    tier.maxSlots = 0
  }

  const statsStore = useStatsStore()
  statsStore.dailyStats.itemsSold++
  return itemId
}
```

## Step 3 — Cập nhật DeliveryManager.ts

### 3A. `handleShelfInteraction` — phân biệt shelfRole

Đây là hàm cốt lõi cần refactor. Thay thế hoàn toàn hàm hiện tại:

```typescript
/**
 * Gọi từ MainScene khi player tương tác [E] với kệ trong lúc đang carry.
 * Phân biệt hai luồng:
 *   - Kệ bán (selling) → openSetPrice
 *   - Kệ kho (storage) → lưu kho, không popup
 * Trả về true nếu đã xử lý interaction.
 */
handleShelfInteraction(shelfId: string): boolean {
  const deliveryStore = useDeliveryStore()
  if (!deliveryStore.carriedBox) return false

  const furnitureStore = useFurnitureStore()
  const shelf = furnitureStore.placedShelves[shelfId]
  
  // Shelf không tồn tại → không xử lý
  if (!shelf) return false

  const carried = deliveryStore.carriedBox

  // Case 1: Furniture box → kích hoạt Build Mode (không phân biệt kệ)
  if (carried.type === 'furniture') {
    furnitureStore.startBuildMode(carried.itemId)
    this.removeCarriedBox()
    deliveryStore.dropBox()
    return true
  }

  // Lấy shelfRole từ data (có migration fallback)
  const shelfRole = shelf.role ?? 'selling'

  if (shelfRole === 'selling') {
    // ====== SELLING SHELF LOGIC ======
    // Bước 1: Điền hàng vào tầng kệ (tự động tìm tầng trống/phù hợp)
    const filled = this.stockItemToSellingShelf(shelfId, carried.itemId, carried.quantity)
    
    if (filled > 0) {
      // Bước 2: Mở popup định giá
      const inventoryStore = useInventoryStore()
      const shopItem = inventoryStore.shopItems[carried.itemId]
      if (shopItem) {
        deliveryStore.openSetPrice({
          shelfId,
          tierIndex: 0, // SetPriceModal không cần tierIndex cụ thể để hiển thị
          itemId: carried.itemId,
          name: shopItem.name,
          imageUrl: carried.imageUrl || '',
          currentPrice: shopItem.sellPrice,
          marketPrice: shopItem.sellPrice,
          buyPrice: shopItem.buyPrice,
        })
      }
    }

    this.removeCarriedBox()
    deliveryStore.dropBox()
    return true

  } else {
    // ====== STORAGE SHELF LOGIC ======
    // Bước 1: Điền hàng vào kho (shopInventory)
    const inventoryStore = useInventoryStore()
    if (!inventoryStore.shopInventory[carried.itemId]) {
      inventoryStore.shopInventory[carried.itemId] = 0
    }
    inventoryStore.shopInventory[carried.itemId] += carried.quantity

    // Bước 2: KHÔNG mở popup định giá
    this.removeCarriedBox()
    deliveryStore.dropBox()
    return true
  }
}

/**
 * Helper: Điền itemId vào tier của kệ bán hàng.
 * Ưu tiên tier đang có cùng itemId còn slot, nếu không thì tìm tier trống.
 * Trả về số lượng đã điền được (0 = kệ đầy).
 */
private stockItemToSellingShelf(shelfId: string, itemId: string, quantity: number): number {
  const furnitureStore = useFurnitureStore()
  const shelf = furnitureStore.placedShelves[shelfId]
  if (!shelf) return 0

  // Tìm tier phù hợp (ưu tiên tier đang có cùng itemId)
  let targetTierIdx = -1
  for (let i = 0; i < shelf.tiers.length; i++) {
    const tier = shelf.tiers[i]
    if (tier.itemId === itemId && tier.slots.length < tier.maxSlots) {
      targetTierIdx = i
      break
    }
  }

  // Không tìm thấy tier cùng loại → tìm tier trống
  if (targetTierIdx === -1) {
    for (let i = 0; i < shelf.tiers.length; i++) {
      if (shelf.tiers[i].itemId === null) {
        targetTierIdx = i
        break
      }
    }
  }

  if (targetTierIdx === -1) return 0 // Kệ đầy

  // Điền quantity đơn vị vào tier
  // Delegate cho furnitureStore để giữ tính nhất quán
  furnitureStore.fillTier(shelfId, itemId, targetTierIdx)
  return quantity
}
```

> ⚠️ **Lưu ý:** `stockItemToSellingShelf` gọi `furnitureStore.fillTier()` là đủ. Hàm `fillTier` đã xử lý việc trừ inventory và điền slot nên không cần làm thêm gì.

```typescript
private updateHintText(playerX: number, playerY: number) {
  const deliveryStore = useDeliveryStore()
  const cam = this.scene.cameras.main

  if (deliveryStore.carriedBox) {
    // Đang cầm → gợi ý F để thả, E để cất vào kệ
    this.hintText
      .setText('[F] Đặt xuống  •  [E] Cất vào kệ')
      .setVisible(true)
      .setPosition(cam.width / 2, cam.height - 80)
      .setOrigin(0.5)
    return
  }

  // Kiểm tra có box gần không (trong 100px)
  let hasNearby = false
  for (const box of this.boxes) {
    if (box.isBeingCarried) continue
    const dist = Phaser.Math.Distance.Between(playerX, playerY, box.sprite.x, box.sprite.y)
    if (dist < 100) {
      hasNearby = true
      break
    }
  }

  if (hasNearby) {
    this.hintText
      .setText('[F] Nhặt thùng hàng')
      .setVisible(true)
      .setPosition(cam.width / 2, cam.height - 80)
      .setOrigin(0.5)
  } else {
    this.hintText.setVisible(false)
  }
}
```

## Ghi chú Kiến trúc

**Tại sao không dùng `inventoryStore.shopInventory` cho Storage Shelf?**

Hiện tại cả 2 loại kệ đều cuối cùng để hàng vào `inventoryStore.shopInventory`. Đây là **intentional simplification** — kho thực tế và kho bày bán đều cùng chung nguồn inventory. Sự phân biệt chỉ ở **khả năng tiếp cận của NPC** và **hành động sau khi stocking** (show/hide SetPrice popup).

Nếu sau này cần tách kho riêng, chỉ cần thêm `storageInventory: Record<string, number>` vào `inventoryStore` và sửa `handleShelfInteraction` để write vào đúng bucket.

**Tại sao hintText ở Phaser thay vì Vue Overlay?**

Vue Overlay yêu cầu reactive watcher theo dõi `deliveryStore.carriedBox` và khoảng cách player tới box (là dữ liệu Phaser). Polling khoảng cách từ Vue không phù hợp. Phaser hintText đơn giản hơn và đảm bảo không nhập nhèm Vue/Phaser reactivity boundary.
